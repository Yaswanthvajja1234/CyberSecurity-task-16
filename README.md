1. Introduction to Incident Response
Incident Response is a structured approach to handling and managing security breaches.
2. Simulated Incident
A simulated brute-force login attempt scenario was analyzed for this task.
3. Identification Phase
System logs were analyzed to detect repeated failed login attempts.
4. Classification
The event was classified as a potential brute-force authentication attack.
5. Containment
Temporary measures were applied to isolate affected accounts.
6. Eradication
Passwords were reset and stronger authentication controls were enforced.
7. Recovery
Systems were restored after confirming removal of suspicious activity.
8. Incident Timeline Documentation
All events and response actions were documented chronologically.
9. Preventive Recommendations
Recommendations included enabling MFA, account lockout policies, and continuous monitoring.
10. Conclusion
This task strengthened practical knowledge of handling and responding to security incidents.
